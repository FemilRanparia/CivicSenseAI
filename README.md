# CivicSense AI - Empowering Voters Through AI 🇮🇳

![CivicSense AI](https://img.shields.io/badge/Status-Live-success) ![Tech Stack](https://img.shields.io/badge/Stack-MERN-blue)

**Live Demo:** [https://civicsenseaielectionassistant.netlify.app](https://civicsenseaielectionassistant.netlify.app)

CivicSense AI is a full-stack, AI-powered election simulation and guidance platform designed to demystify the Indian voting process for first-time and experienced voters alike.

## 🚀 Overview
In India, millions of first-time voters become eligible every year, yet the sheer complexity of registration, EVM operation, and booth protocols leads to voter apathy or confusion. CivicSense AI solves this by transforming static government guidelines into a highly personalized, interactive, and simulated educational journey.

This project sits at the intersection of **Civic Technology** and **Educational Technology**, leveraging Generative AI to guide users step-by-step through their democratic duties.

## 🧠 Core Features & Technical Implementations

### 1. Zero-Friction "Prompt Injection"
Instead of building a generic chat wrapper where the user has to write complex prompts, CivicSense AI uses **context-aware application state**. Upon signup, the system captures demographic data (Age, State, First-Time Voter status). The backend automatically injects this profile into a heavily engineered system prompt before sending it to the Gemini API, guaranteeing a perfectly tailored response without the user doing any heavy lifting.

### 2. Gamified Journey Timeline
Complex bureaucratic steps are broken down into a "Journey Timeline"—a visual, interactive checklist generated by the AI based on the user's age and state (e.g., Form 6 registration for 18-year-olds vs. Roll verification for 30-year-olds). Users can track their real-world progress visually.

### 3. Multi-Model Fallback Loop
To guarantee 100% uptime, the backend is engineered with a robust try/catch fallback loop. If the primary Gemini model (`gemini-flash-lite-latest`) is overloaded or rate-limited (a common 503 error), the server instantly catches the error and seamlessly degrades to alternative models (`gemini-2.0-flash-lite-001`, `gemini-pro-latest`, etc.) ensuring the user never sees an error.

### 4. Payload Trimming Architecture
To prevent `413 Payload Too Large` crashes common in LLM chat apps, the React frontend is heavily optimized. Instead of sending the massive, ever-growing user profile and chat history on every request, the frontend strips the payload down to essential demographic bytes (Age, State, FirstTimeVoter status). This guarantees lightning-fast network requests.

## 🚀 Local Setup & Deployment

### Prerequisites
* Node.js (v18+)
* MongoDB Atlas Cluster URI
* Google Gemini API Key

### Local Development
1. Clone the repository: `git clone https://github.com/FemilRanparia/CivicSenseAI.git`
2. **Backend Setup:**
   * `cd backend`
   * `npm install`
   * Create a `.env` file with `PORT=5000`, `GEMINI_API_KEY=...`, `MONGODB_URI=...`
   * `npm run dev`
3. **Frontend Setup:**
   * `cd ../frontend`
   * `npm install`
   * Create a `.env` file with `VITE_API_URL=http://localhost:5000/api`
   * `npm run dev`

### Production Deployment
* **Backend (Render):** Deploy the `backend` directory as a Node Web Service. Add environment variables.
* **Frontend (Netlify):** Deploy the `frontend` directory. The included `netlify.toml` automatically configures the build settings (`npm run build`, publish directory `dist`) and handles React Router redirects. Set `VITE_API_URL=https://<your-render-url>/api`.
